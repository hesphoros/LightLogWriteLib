Detailed Description

Implementation of the LightLogWrite class.

This class provides a thread-safe implementation for writing logs to a file.

Constructor for LightLogWrite_Impl.

Parameters
maxQueueSizeThe max log written queue size
strategyThe strategy for handling full log queue
reportIntervalThe interval for reporting log overflow

This constructor initializes the log writer with a specified maximum queue size, a strategy for handling full queues, and an interval for reporting log overflow.

Note
The default maximum queue size is 500000, the default strategy is to block when the queue is full, and the default report interval is 100 discarded logs.

Sets the directory and base name for lasting logs.

Parameters
sFilePathThe directory path where the logs will be stored
sBaseNameThe base name for the log files

This function sets the directory and base name for lasting logs.

  • It will create a new log file based on the current date and time.
  • The log files will be named in the format "BaseName_YYYY_MM_DD_AM/PM.log". such as "LightLogWriteImpl_2025_05_27_AM.log".
  • If the directory does not exist, it will be created.
    Note
    This function should be called before writing any logs to ensure that the logs are stored correctly.
